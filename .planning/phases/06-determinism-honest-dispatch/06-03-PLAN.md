Slice 3 of Phase 6 (vertical, MVP): harden the GPU forest to byte-reproducible determinism, then measure the cost. The slice runs in the locked order audit → fix → gate → measure. First (front-loaded per D6-1) a non-determinism AUDIT enumerates every remaining GPU non-determinism source in the shipped Phase-4/5 train path and pins a fix for each. Then the GPU scheduler + arena are constrained under `deterministic=true` (fixed tree-index build/merge order, deterministic buffer reuse, no free multi-stream overlap, no residual float reduction). Then the DET-01 gate — two same-seed `deterministic=true` GPU fits produce byte-identical serialized `ForestIR` (exact serde_json string equality, never allclose) for ET+RF × clf+reg — proves it, with `racecheck` as the non-determinism canary. Finally DET-02 measures the Sylva-deterministic-vs-Sylva-fast overhead, and a `checkpoint:human-verify` task qualitatively confirms the external determinism gap (cuML RF / LightGBM-GPU are not byte-reproducible) without any speed claim.

Purpose: DET-01 (byte-reproducibility) is a headline correctness guarantee and the determinism wedge; DET-02 documents its cost honestly. Integer histograms make values order-free, but *ordering* (streams, arena reuse, tree-offset assignment) and any residual float reduction can still leak run-to-run non-determinism — the audit + fixes + byte gate close exactly that gap.

Output: `06-NONDET-AUDIT.md` (the enumerated sources + fixes), deterministic-ordering changes in `scheduler.rs`/`arena.rs`, the Rust `deterministic_cpu_gpu.rs` two-run byte gate + the Python equivalent, and `determinism_overhead.py` (overhead bench + qualitative external-gap section). After this plan, the DET-01 byte gate is green under `deterministic=true`, the deterministic path is `racecheck`-clean, and the overhead number is reported.

This plan implements orchestrator-locked Assumptions D6-1 (front-loaded non-determinism audit), D6-2 (`deterministic=false` FAST mode for a real overhead number, ~95-98% retention target, or documented overhead≈0 with reason), D6-3 (`deterministic` default = True), and D6-6 (external determinism-gap study behind a blocking human-verify checkpoint; gates do NOT depend on it).

## Phase Goal

**As a** ML practitioner using Sylva from Python, **I want to** get a byte-identical model from two same-seed `deterministic=True` GPU fits and see the documented overhead of that guarantee, **so that** I can reproduce, diff, and certify trained models exactly — a guarantee cuML RF and LightGBM-GPU do not offer.

- **D6-1 (LOCKED):** front-load a non-determinism AUDIT task — the first determinism task enumerates remaining GPU non-determinism sources in the shipped Phase-4/5 train path (multi-stream completion order, arena buffer-reuse order, tree→node-offset assignment, any residual float reduction) and pins each fix.
