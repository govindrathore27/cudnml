## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| crate registries -> local build | Pinned crates (cudarc, pyo3) downloaded over TLS and compiled locally; Cargo.lock pins exact versions |
| toolchain installers -> dev host | rustup, VS Build Tools, maturin, CUDA toolkit downloaded over TLS from vendor sources |

This is a THROWAWAY, LOCAL, single-developer CUDA toolchain spike: no network services, no user-supplied input, no authentication, no persisted secrets, no PII. The only realistic surface is supply-chain integrity of pinned dependencies and TLS-downloaded toolchain installers.

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-01 | Tampering | crate dependency set (cudarc, pyo3) | mitigate | Pin exact versions in Cargo.toml + commit Cargo.lock; deps legitimacy-audited in 01-RESEARCH.md "Package Legitimacy Audit" (all Approved) |
| T-01-02 | Spoofing | toolchain installers (rustup / VS BT / maturin) | accept | Standard vendor TLS download on a single dev host; out of scope to re-verify installer signatures for a throwaway spike — low value, local-only |
| T-01-SC | Tampering | cargo/pip installs in this plan | mitigate | No `[ASSUMED]`/`[SUS]` packages introduced here (cudarc/pyo3 are Approved in the audit; maturin/cupy SUS verdicts were cleared as false positives in 01-RESEARCH.md). No legitimacy human-checkpoint required. |

There are NO high-severity threats in this plan and none can be honestly justified: no network listener, no untrusted input, no secret material. ASVS L1 categories V2/V3/V4/V6 are N/A; V14 (supply chain) is satisfied by pinning + the legitimacy audit.
