    Based on the Wave-0 spike result recorded in `09-01-SUMMARY.md`, finalize `export_to_treelite()` in `python/sylva/export.py`. The following are non-negotiable implementation details regardless of what the spike found:

    Task type determination from the serde-serialized `ir["task"]` field: the Rust `Task` enum serializes in two possible ways via serde: either `{"Classification": {"n_classes": N}}` (when `n_classes` is a named field) or just `"Regression"`. Check `isinstance(ir["task"], dict)` to distinguish. Extract `n_classes = ir["task"]["Classification"]["n_classes"]` for classifier; `n_classes = 1` for regressor. Then: `is_clf = n_classes >= 2` (regression has n_classes=1 per ForestIR.n_classes() method in ir.rs lines 74-79).

    `ModelBuilder` instantiation — these parameters are binding per D-03:
    - `threshold_type="float32"`, `leaf_output_type="float32"` — matches ForestIR f32 fields.
    - `metadata=Metadata(num_feature=ir["n_features"], task_type="kMultiClf" if is_clf else "kRegressor", average_tree_output=True, num_target=1, num_class=[n_classes] if is_clf else [1], leaf_vector_shape=(1, n_classes) if is_clf else (1, 1))`.
    - `tree_annotation=TreeAnnotation(num_tree=ir["n_trees"], target_id=[0]*ir["n_trees"], class_id=[-1]*ir["n_trees"] if is_clf else [0]*ir["n_trees"])`.
    - `postprocessor=PostProcessorFunc(name="identity")` — D-03 locked, spike confirmed.
    - `base_scores=[0.0] * (n_classes if is_clf else 1)`.

    Tree walk — mandatory per D-04:
    - `tree_offsets = ir["tree_offsets"]` (list of ints).
    - For `t` in `range(ir["n_trees"])`: `start = tree_offsets[t]`; `end = tree_offsets[t+1]`; `builder.start_tree()`; for `global_id` in `range(start, end)`: `local_id = global_id - start`; call `builder.start_node(local_id)`; if `ir["is_leaf"][global_id]` is True, call the leaf path; else call the numerical_test path with `left_child_key = ir["left_child"][global_id] - start` and `right_child_key = ir["right_child"][global_id] - start`; `builder.end_node()`; `builder.end_tree()`.

    Leaf path (is_clf, multi-class n_classes >= 2): `o = ir["leaf_offset"][global_id]`; `nc = n_classes`; `proba = ir["leaf_proba"][o * nc : (o + 1) * nc]`; `builder.leaf(list(proba))`.
    Leaf path (regressor n_classes == 1): `o = ir["leaf_offset"][global_id]`; `builder.leaf(float(ir["leaf_value"][o]))`.

    Internal node path: `lc = ir["left_child"][global_id] - start`; `rc = ir["right_child"][global_id] - start`; `default_left = (ir["default_child"][global_id] == ir["left_child"][global_id])`; `builder.numerical_test(feature_id=int(ir["feature_id"][global_id]), threshold=float(ir["threshold"][global_id]), default_left=default_left, opname="<", left_child_key=lc, right_child_key=rc)`. Optionally call `builder.data_count(int(ir["node_sample_count"][global_id]))` if the key exists; omit silently if not (use `.get()`).

    After `builder.end_tree()` for all trees, `return builder.commit()`.

    Wrap the loop body in `try/except (KeyError, IndexError) as e: raise ValueError(f"export_to_treelite: malformed ForestIR: {e}") from e` and `except Exception as e: raise RuntimeError(f"export_to_treelite: ModelBuilder error: {e}") from e`.

    Run `python -m pytest python/tests/test_export.py -x -q` after each fix iteration. Common failure modes and their fixes (per RESEARCH.md Pitfalls):
    - gtil parity mismatch with values near 0.6-0.7 for what should be 0.9: wrong postprocessor (sigmoid applied to probabilities) — ensure `postprocessor="identity"`.
    - NaN or index error from gtil: global IDs passed to child_key — re-verify the `- start` reindex is applied on BOTH left and right child keys.
    - Shape mismatch in assert_allclose: check `treelite.gtil.predict()` return shape against `sylva predict_cpu` return shape; gtil may return `(n, n_classes)` or `(n,)` for binary — adjust the test comparison (squeeze or expand as needed).

    Update `test_export.py` to fix any test assertions that fail due to shape differences discovered during the spike (e.g., squeeze reg output).
