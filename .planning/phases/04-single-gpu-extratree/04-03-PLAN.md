<objective>
Close the phase: prove the GPU ExtraTree equals the CPU oracle **bit-exactly** for both
classifier and regressor (GPU-02), extend `CudaBackend::fit` to the regressor, make every ET
kernel `compute-sanitizer`-clean across all four tools, and stand up the Comparative Baseline
Study harness (distributional vs sklearn + transfer-inclusive timing, both informational).
The bit-exact byte-compare and the four-tool sanitizer-clean are the non-negotiable phase
gate.

Purpose: GPU-02 is the load-bearing gate of this phase — the GPU tree must reproduce the
shipped CPU oracle's `ForestIR` byte-for-byte on a fixed seed, and every kernel must be free
of memory/race/sync/init hazards. The regressor closes GPU-01's "classifier + regressor"
scope. The Python study satisfies the phase's Comparative Baseline Study (correctness parity
primary; single-tree timing reported, never an end-to-end speed claim).

Output: the regressor path in `fit`/`assemble`, `parity_cpu_gpu.rs` (clf + reg byte-compare
gate), `sanitizer_et_kernels.rs` (per-kernel single-launch targets), and the
`python/tests/gpu_parity/` study harness, ending with the blocking human-verify checkpoint
that runs all four sanitizer tools on the dev host.
