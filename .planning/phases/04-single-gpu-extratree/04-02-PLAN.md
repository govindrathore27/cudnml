<objective>
Deliver the first WORKING GPU ExtraTree: `CudaBackend::fit` trains a single ExtraTree
**classifier** end-to-end on the GPU and returns a structurally valid `ForestIR` that
`predict` (reusing the CPU traversal) can consume. This is the thinnest real vertical slice —
after this plan a user can call `CudaBackend.fit(x, y, cfg)` for an ET classifier and get a
forest back, built breadth-first with a privatized integer histogram and a raw-range Philox
threshold (Strategy A), scored host-side from device counts.

Purpose: This is the GPU-01 capability — privatized shared-memory histogram kernel + fused
random-candidate split + scatter-partition, breadth-first. Bit-exactness against the CPU
oracle (GPU-02) and the regressor + sanitizer gate are Plan 03; this plan makes the path
real and runnable for the classifier. Keeping scoring host-side (calling the shipped
`cpu::criterion` functions on exact integer counts) is what makes Plan 03's bit-exact gate
reachable without device-float divergence.

Output: the four-file kernel + launch-wrapper set (`kernels.rs` ET source, `histogram.rs`,
`partition.rs`, `split_score.rs`), the breadth-first `scheduler.rs`, fit-scoped
`device_buffers.rs`, `assemble.rs`, and a wired `CudaBackend::fit`/`predict`.

## Assumptions (locked architecture — see Plan 01 for the full list with alternatives)

- **Strategy A (LOCKED):** raw-range Philox threshold + raw `x <= threshold` partition;
  privatized histogram = integer counting engine only; CPU oracle untouched. (Alt: B binned
  split / C change oracle — rejected.)
- **Bin contract = upload bins too (LOCKED):** upload BOTH raw X and the Phase-3 BinnedMatrix
  and run the histogram over the data, to exercise the Phase-3→4 contract early and de-risk
  Phase 5. (Alt: raw-X-only thinnest slice — noted.)
- **Scoring = host-side from device integer counts (LOCKED):** call `cpu::criterion` on the
  returned counts; on-device float scoring is deferred. (Alt: on-device `-fmad=false` score —
  noted; deferred to Phase 5.)
- **Frontier→block mapping:** one block per frontier node; node arrays pre-sized to
  `2^(max_depth+1)-1`. Arena pool (GPU-05) + small-node CPU cutover (GPU-06) are Phase 5.
</objective>
