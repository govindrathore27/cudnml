<objective>
Lay the parity spine for the GPU ExtraTree: make sylva-cuda depend on sylva-core, add the
`-fmad=false` bit-parity compile flag to every NVRTC compile, and prove the shared Philox
RNG stream is bit-identical on-device BEFORE any tree is ever built. This is the first
working end-to-end GPU capability of the phase: an on-device Philox kernel that reproduces
the three frozen KAT vectors. After this plan, `CudaBackend::new()` exists and gates its own
construction on the on-device KAT passing.

Purpose: The entire phase gate (GPU == CPU oracle bit-exact, GPU-02) is unreachable unless
the GPU draws the *identical* random stream the CPU draws. The cheapest possible proof of
that is an on-device KAT check against `rng/kat.rs` — done first so every later kernel builds
on a verified RNG. The `-fmad=false` flag (locked NVRTC decision) and the sylva-core
dependency are the other two prerequisites every later plan needs.

Output: `CudaBackend` skeleton + `philox_device.rs` (CUDA-C Philox source + on-device KAT
verifier) + the `philox_device_kat.rs` integration test + the `-fmad=false` flag on
`compile_for_sm89`.

## Assumptions (locked architecture for this phase — record so it can be revisited)

This phase has no CONTEXT.md; the orchestrator resolved the load-bearing forks. Treat as
LOCKED, with alternatives noted:

1. **Split strategy = Strategy A (LOCKED):** GPU replays the shipped CPU oracle bit-for-bit
   — random threshold from the RAW f32 `[min,max]` range via the shared Philox stream,
   partition on raw `x <= threshold`; the privatized histogram is an *integer counting
   engine* only. The Phase-2 CPU oracle (`split_et.rs`) is NOT modified. (Alternatives: B =
   binned-candidate split; C = change the CPU oracle to bin-split — both mutate verified
   Phase-2 behavior. Rejected.)
2. **NVRTC flag = `-fmad=false` on every compile, never `--use_fast_math` (LOCKED):**
   carried from the Phase-3 no-FMA/no-fast-math parity metadata. (Plan 01 implements this.)
3. **Philox device replay (LOCKED):** inline the shipped Philox-4x32-10 in-kernel keyed by
   the exact `(seed, tree, node, feature, draw)` counter packing, KAT-tested on-device
   against `rng/kat.rs` BEFORE building any tree. (This plan is that proof.)
</objective>
