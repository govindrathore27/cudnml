<objective>
Close the oracle's trust loop against the real reference: a minimal test-only PyO3 seam exposing `CpuBackend` to Python, an empirical calibration step that sets the parity bar from the sklearn-vs-sklearn null spread, and the distributional parity harness that gates CpuBackend against scikit-learn ET/RF (accuracy/proba CI + KS on split statistics) like-for-like — the SC-6 Comparative Baseline Study gate — plus informational-only CPU timing (SC-7). This is the differential side of EST-07 and the documented sklearn-equivalence half of ENG-04.

Purpose: The proptest invariants (Plan 04) prove internal consistency; this plan proves external CORRECTNESS — that Sylva's ET/RF are statistically indistinguishable from sklearn's. Because sklearn's serial PRNG is un-replayable (per the parity contract), the bar is distributional and must be calibrated: measure how much sklearn disagrees with ITSELF across seeds, then require Sylva to sit inside that null band. The minimal PyO3 seam keeps the harness in Python (where sklearn + scipy live) without building the full estimator API (deferred to Phase 5). All comparisons are like-for-like (ET-vs-ET, RF-vs-RF); timing is reported, never gated — the binding fairness protocol forbids an end-to-end speed claim at a foundational phase.

Output: `pyseam.rs` (test-only `fit_cpu`/`predict_cpu`/`split_statistics`), the maturin/pyproject wiring for it, `python/tests/parity/` (conftest, datasets, calibration → `thresholds.json`, the distributional gate + informational timing).
</objective>
