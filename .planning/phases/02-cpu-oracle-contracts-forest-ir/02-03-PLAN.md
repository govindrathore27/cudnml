<objective>
Second runnable slice: extend `CpuBackend` to train RandomForest (classifier + regressor) by adding bootstrap row resampling and an exact best-split (sorted-midpoint) splitter, reusing the recursive builder, criterion, ForestIR, and NaN-safe predict from Plan 02 (ENG-03 RF portion). This completes SC-2 — both ExtraTrees AND RandomForest, classifier + regressor each, trained correctly this phase per locked decision D-02.

Purpose: RandomForest differs from ExtraTrees in exactly two places: (1) per-tree bootstrap resampling (n rows with replacement) instead of using all rows, and (2) an exact best-split search (sort each candidate feature's node values, evaluate impurity at midpoints between consecutive distinct values, pick the best) instead of one random threshold. Everything else — the recursive builder, impurity criterion, ForestIR layout, default_child policy, NaN-safe predict, determinism strategy — is shared. Building RF as a thin extension of the ET slice keeps the oracle's two algorithms consistent and avoids divergent predict paths.

Output: the RF best-split splitter, the bootstrap sampler, `build_node` dispatch on `cfg.algo`, and `CpuBackend::fit` honoring `bootstrap` — all `cargo test -p sylva-core` green, completing the four-estimator (ET/RF × clf/reg) oracle.
</objective>

    - On a node where a feature linearly separates two classes, the RF splitter picks a midpoint threshold between the two class clusters (the exact best split), yielding the maximal impurity improvement.
    - Candidate thresholds are midpoints between consecutive DISTINCT sorted values only (no threshold equals a data value; duplicates collapsed).
    - Pairs of consecutive values closer than `FEATURE_THRESHOLD` are skipped.
    - Tie-break: equal-improvement splits resolved by lower `(feature_id, threshold)` (same total order as ET).
    - `default_child` = higher-row-count child; tie → left (same D-01 policy as ET).
  </behavior>
