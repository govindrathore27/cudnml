<objective>
Stand up the device-neutral contract layer for Phase 2: a new CUDA-free `sylva-core` crate holding the `trait Backend` seam (ENG-01), the consumer-complete SoA `ForestIR` (ENG-02), the `TrainConfig`/error types, and the stateless Philox-4×32-10 RNG with human-verified known-answer-test vectors (ENG-06). This is the foundation every later Phase-2 slice (ET training, RF training, invariants, parity harness) imports.

Purpose: These three contracts are the "near-rewrite-if-deferred" CPU↔GPU parity surface flagged in STATE.md. The trait must keep CUDA out of the boundary so Phase 4's `CudaBackend` is purely additive; the IR must be designed for all known consumers now (train/predict + SHAP cover counts + Treelite-export fields per D-03) so it is never rewritten; the Philox stream must ship documented, verified KAT vectors so Phase 4's CUDA reimplementation is bit-verifiable.

Output: `crates/sylva-core` crate (compiles, `cargo test -p sylva-core` green), `trait Backend` + `trait HistogramBackend`, `ForestIR` (SoA + serde), `TrainConfig`/`Criterion`/`MaxFeatures`/`Task` types, `SylvaError`, `philox4x32_10` + conversion + counter packing, and frozen+verified `rng/kat.rs`.
</objective>

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01 | Tampering | `TrainConfig` / `ArrayView` boundary | mitigate | `TrainConfig::validate` + dim checks reject malformed params/shapes with typed `SylvaError` (no panic). |

| T-02-03 | Information disclosure | Philox RNG misuse | accept | Philox is a statistical, NON-cryptographic RNG; documented as non-CSPRNG so it is never mistaken for secure randomness. No secrets in this phase. |
