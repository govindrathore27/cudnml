<objective>
First runnable end-to-end slice: a pure-Rust `CpuBackend` (ndarray + rayon) that trains an ExtraTrees forest (classifier + regressor) and predicts from the resulting `ForestIR`, with NaN/missing-value default-direction routing (ENG-03 ET portion, ENG-05). After this plan, a real user can `CpuBackend::fit` ET data and `predict` on it â€” the first exercisable trainâ†’predict path and the spine the RandomForest slice (Plan 03) extends.

Purpose: This is the trusted CPU oracle's ExtraTrees path â€” the simplest splitter (random thresholds) first, per the project's "single tree before forest, ET before RF" ordering. The recursive node builder, criterion (Gini/entropy/MSE), predict traversal, and NaN routing built here are shared by RF in Plan 03. Determinism is by construction: parallelize across trees with rayon (each tree's draws are a pure function of `tree` via Philox keying), build each tree single-threaded with fixed-order f32 accumulation so the model is byte-reproducible and the future GPU==CPU bit-exact gate is reachable.

Output: `CpuBackend` (`impl Backend`), the ET random-threshold splitter, the impurity criterion, the recursive forest builder, and NaN-safe predict â€” all `cargo test -p sylva-core` green incl. NaN fixtures.
</objective>
