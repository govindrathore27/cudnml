<objective>
Run the project's FIRST real end-to-end speed comparison honestly. Build the Comparative Baseline
Study harness that times `fit(X, y)` from a host numpy array (dtype coercion + H2D + quantization
included) for Sylva ET/RF vs sklearn ET/RF (n_jobs=-1), cuML RF (labeled a different algorithm),
and XGBoost rf-mode, on a large dense dataset — with accuracy reported beside every speed cell,
cold/warm separated, versions pinned, like-for-like enforced, and OOM/failures reported honestly.
Accuracy parity is the gate; speed is reported with the Phase-7-crossover caveat.

After this plan there is a reproducible study script + a pinned manifest + an automated
accuracy-parity gate, and a recorded honest result (including cuML availability on the host).

Purpose: This is study SC-6/SC-7 — the per-phase data point that FEEDS the authoritative Phase-7
crossover. The fairness protocol is binding (STATE.md blocker; .planning/research/PITFALLS.md
Pitfalls 1, 2, 7, 13). cuML on native Windows is historically unavailable, so the cuML baseline is
gated behind a blocking human-verify checkpoint (WSL2 path OR honest "unavailable"). The estimators
this study benchmarks come from Plan 05; the kernels it exercises end-to-end are GPU-03 (forest) +
GPU-04 (RF) — this plan reports their real-world speed under the fairness protocol.

Output: `comparative_study.py` (harness), `study_manifest.py` (pins), `test_study_accuracy_parity.py`
(the accuracy gate), and the cuML-availability checkpoint.

## Assumptions (locked decisions — revisitable; each carries its alternative)

- **Comparative study scope (LOCKED, locked-decision-6):** end-to-end-from-numpy fit time, Sylva ET
  vs sklearn ET (n_jobs=-1), Sylva RF vs sklearn RF + cuML RF (labeled a different algorithm) +
  XGBoost rf-mode; accuracy parity is the GATE, speed REPORTED with the Phase-7 crossover caveat;
  cold/warm separated, transfers included, versions pinned, OOM/failures reported honestly;
  like-for-like enforced. Follows .planning/research/PITFALLS.md. (Alternative: a speed-gated study —
  rejected; Phase 7 is the authoritative speed gate, not this phase.)
- **cuML baseline gated behind human-verify (LOCKED, OQ7):** cuML-on-Windows may be unavailable; a
  blocking `checkpoint:human-verify` resolves install/WSL2 vs honest "unavailable". XGBoost has
  Windows wheels and should run natively. (Alternative: assume cuML present — rejected; would fake or
  crash on the Windows host.)
- **ET GPU baseline = none (LOCKED, A5):** cuML ships no GPU ExtraTrees as of the research check, so
  Sylva ET is compared only to sklearn ET; this is stated in the report, not hidden. (Alternative:
  add a cuML ET baseline if cuML ships one — note the kill-risk if so.)
