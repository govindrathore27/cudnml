<objective>
Land the two device-neutral Rust foundations the rest of Phase 5 builds on, with zero GPU
or Python dependency so this plan runs first and unblocks every wave: (1) extend `TrainConfig`
with `max_samples` and `class_weight` end-to-end at the config layer (EST-03), and (2) add
`crates/sylva-core/src/importance.rs` computing real MDI `feature_importances_` from a
`ForestIR` (EST-04 — the direct cuML gap).

After this plan: a fitted forest's IR can be turned into a correct, sklearn-shaped MDI vector,
and the config surface carries the two parameters the Python estimators and the RF
bootstrap/weighted path (Plan 03) and the estimator seam (Plan 05) consume.

Purpose: This is the smallest vertical slice that adds NEW user-observable capability
(importances over the existing CPU-trained IR) while preparing the config contract every later
plan reads. It is intentionally GPU-free so it has no Phase-4 dependency and can validate on
GPU-less CI.

Output: extended `config.rs` (two enums + two fields + validation), new `importance.rs` (MDI),
and the `lib.rs` module wiring + re-export.

## Assumptions (locked decisions — revisitable; each carries its alternative)

- **D-05 max_samples + class_weight in TrainConfig (LOCKED):** add both end-to-end starting
  here at the Rust config layer. `max_samples` semantics mirror sklearn: `None` = n rows,
  `Fraction(f in (0,1])`, `Count(usize)`; only meaningful when `bootstrap=true`. `class_weight`
  is `Balanced | Map(class_id→weight)`; it folds into per-row sample_weight at fit (Plan 03).
  (Alternative rejected: keep config minimal and pass these only through Python — makes the Rust
  RF/weighted path and the bit-exact gate unable to see them; rejected.)
- **MDI in f64 internally, cast to f32 (LOCKED):** matches sklearn's double-precision recipe;
  distributional parity is the realistic bar (ENG-04). (Alternative: exact f32 accumulation —
  noted; only needed if a future gate requires bitwise f32 MDI.)
</objective>
