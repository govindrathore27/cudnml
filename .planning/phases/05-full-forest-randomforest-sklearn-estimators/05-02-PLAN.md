<objective>
Scale the Phase-4 single GPU ExtraTree into a full GPU FOREST: loop the breadth-first
single-tree builder over `n_estimators`, give each tree its per-tree Philox/bootstrap schedule
(reusing the shipped `bootstrap_indices`), add the integer sibling-histogram-subtraction kernel
(the forest's key performance win), and assemble the per-tree fragments into one multi-tree
`ForestIR` with correct global offsets (GPU-03).

After this plan a user can call `CudaBackend.fit(x, y, cfg)` with `n_estimators > 1` for
ExtraTrees (classifier and regressor) and get back a forest that is byte-identical to the CPU
oracle on a fixed seed — the thinnest end-to-end forest slice.

Purpose: This is the GPU-03 capability and the foundation Plan 03 (RF best-split) and Plan 04
(arena + cutover) extend. Sibling subtraction is integer-only specifically so determinism and
exactness hold (DET-01 foundation); the bootstrap reuses the shipped Philox so RF's eventual
bit-exact gate (Plan 03) is reachable. RandomForest's binned best-split, sample_weight, the
dedicated arena, and the CPU cutover are later plans — this plan delivers the forest scheduler
and the bit-exact ET forest.

Output: `forest.rs` (tree loop + per-tree bootstrap + multi-tree assemble), the
`SIBLING_SUBTRACT_SRC` kernel + `launch_sibling_subtract` wrapper, the multi-tree `assemble.rs`
global-offset logic, the wired `CudaBackend::fit` forest path, and the `forest_cpu_gpu.rs`
bit-exact gate.

## Assumptions (locked decisions — revisitable; each carries its alternative)

- **Sibling subtraction = integer counts only (LOCKED, RESEARCH Pitfall 2 / Pattern 2):** compute
  the smaller child's count histogram directly, derive the larger by `parent − smaller`
  element-wise in u32. (Alternative rejected: subtract weighted FLOAT histograms — non-associative,
  breaks determinism and exactness.)
- **Per-tree bootstrap = shipped bootstrap_indices (LOCKED, Pattern 3):** the host computes each
  tree's row set with `bootstrap_indices(n_rows, seed, tree_id)`; the kernel reads the resulting
  row_index buffer. (Alternative: inline the Philox bootstrap on-device with `node =
  BOOTSTRAP_NODE_SENTINEL` — noted; only needed if host bootstrap dominates the profile, a Phase-7
  concern, not a Phase-5 correctness need.)
- **Reuse Phase-4 FitBuffers (LOCKED for this plan):** no new allocator here; buffers are
  fit-scoped and reused across trees/waves. The stream-ordered/safe-slab arena (GPU-05) is Plan 04.
</objective>
