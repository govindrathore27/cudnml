<objective>
Land the performance + robustness layer that makes the forest production-shaped without changing
what it trains: (1) a fit-scoped `FitArena` that pre-sizes device buffers once with the cudarc
SAFE API and reuses them across every wave and tree — no per-node allocation (GPU-05), and (2) a
small/deep-node CPU leaf-finishing cutover that reuses the shipped CPU builder (GPU-06). Then prove
the new kernels are `compute-sanitizer` clean across all four tools on a forest fit.

After this plan, large forests train without per-node `cudaMalloc` churn, tiny/deep nodes finish
on the CPU (avoiding GPU underutilization), and the new RF/forest kernels are sanitizer-clean —
while the ET and RF bit-exact gates still pass byte-for-byte.

Purpose: GPU-05 and GPU-06 are the two remaining GPU-training requirements. The arena uses the
SAFE slab (locked-decision-1) so there is zero raw-FFI / `unsafe` surface — the literal
`cudaMallocAsync` pool is deferred to a later optimization. The CPU cutover reuses
`sylva_core::cpu` so it cannot diverge from the GPU result. The sanitizer gate is the correctness
proof for every new kernel introduced in Plans 02-03.

Output: `arena.rs` (FitArena safe slab), `cpu_cutover.rs` (CPU leaf-finishing), arena-backed
`device_buffers.rs`, the scheduler wired to draw from the arena, and `sanitizer_rf_kernels.rs`
standalone targets + a documented four-tool sanitizer run.

## Assumptions (locked decisions — revisitable; each carries its alternative)

- **Arena = safe pre-sized fit-scoped slab (LOCKED, locked-decision-1):** use cudarc safe
  `alloc_zeros` buffers pre-sized at fit start and zeroed/reused across breadth-first waves — NO
  per-node allocation, ZERO `unsafe`. (Alternative rejected for now: a raw
  `cuMemAllocAsync`/`driver::sys` stream-ordered pool — defer to a later optimization behind a
  feature flag if the slab's peak memory is too high for very large datasets; it materially changes
  the `unsafe`/sanitizer surface, so it is out of scope this plan.)
- **CPU cutover = reuse sylva_core::cpu builder (LOCKED):** below a documented row CUTOFF (or beyond
  a depth bound), finish the subtree on CPU via the shipped builder so the result is identical to an
  all-GPU build. (Alternative: a separate GPU small-node kernel — more code, more divergence risk;
  rejected for the MVP.)
</objective>
