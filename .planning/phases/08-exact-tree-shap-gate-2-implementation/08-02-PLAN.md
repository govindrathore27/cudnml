    Define PathEntry struct with fields feature_idx: i32, zero_fraction: f64, one_fraction: f64, pweight: f64. Use f64 throughout (per D-04 — f64 accumulation). Add #[derive(Clone, Debug)].

    Implement extend_path(path: &mut Vec<PathEntry>, zero_fraction: f64, one_fraction: f64, feature_idx: i32): following the Lundberg 2020 EXTEND procedure — push a new PathEntry; then walk backwards updating each existing entry's pweight. The EXTEND procedure adjusts pweight for each path entry as described in Algorithm 2 of Lundberg 2020 Nature MI supplementary: for entries i from path.len()-2 down to 0, pweight[i+1] += one_fraction * pweight[i] * (i+1) as f64 / (path.len() as f64); pweight[0] *= zero_fraction. The new entry at position len-1 gets initialized with the passed fractions and pweight = 1.0 if len-1 == 0 else 0.0 (root slot).

    Implement unwind_path(path: &mut Vec<PathEntry>, feature_idx: i32): find the last index where feature_idx matches, reverse the EXTEND operation on remaining entries, then remove that entry.

    Implement sum_path_unwind_weights(path: &[PathEntry], path_index: usize) -> f64: compute the partial sum needed for UNWIND at path_index per the Lundberg formula — this is the "W" weight that multiplies (one_fraction - zero_fraction) * leaf_value.
