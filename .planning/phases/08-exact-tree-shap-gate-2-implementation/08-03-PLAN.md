    4. Add py_ir_to_shap_dict function with #[pyfunction] #[pyo3(name = "ir_to_shap_dict")] signature:
       py_ir_to_shap_dict(py: Python<'_>, ir_handle: &str) -> PyResult<PyObject>
       This converts a ForestIR JSON to a shap custom-dict list: Vec<dict> with keys matching shap's {"trees": [{"children_left": [...], "children_right": [...], "children_default": [...], "features": [...], "thresholds": [...], "values": [...], "node_sample_weight": [...]}]}. Per the shap custom tree format docs:
       - children_left: for each node, left_child (or -1 for leaves, matching shap's TREE_LEAF sentinel -1)
       - children_right: right_child (or -1)
       - children_default: default_child (or -1 for leaves)
       - features: feature_id (LEAF_FEATURE = -2 in shap convention; IR uses -1, map -1 → -2)
       - thresholds: threshold (float, any value for leaves)
       - values: leaf or internal values. For regression: shape [n_nodes, 1, 1] with leaf_value at leaf_offset for leaves, 0.0 for internals. For classification: shape [n_nodes, 1, n_classes] with leaf_proba at leaf_offset, zeros for internals.
       - node_sample_weight: node_weighted_count as f64 (shap uses f64 here).
       Build each tree dict as a PyDict via PyDict::new and insert numpy arrays via ndarray → into_pyarray. Return a Python list of dicts via PyList::new.
       Note: the LEAF sentinel in shap is -2 (not -1). Feature IDs at leaf nodes must use -2 in the output dict. The shap TreeExplainer identifies leaves by children_left[node] == TREE_LEAF (-1 in older shap, -2 in 0.52.x — verify in the validation harness and use the correct sentinel).

    5. Register both functions in sylva_core_pyseam module fn: module.add_function(wrap_pyfunction!(py_shap_values, module)?)?; and module.add_function(wrap_pyfunction!(py_ir_to_shap_dict, module)?)?;.
