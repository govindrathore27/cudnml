<objective>
Build the standalone CPU `Quantizer`: the thin end-to-end slice that turns a dense f32 matrix into a SoA `BinnedMatrix` (uint8/uint16) via exact per-feature quantile bins, with a reserved missing bin and a minimal `QuantizeReport`. This is the binning front-end every Phase-4 GPU histogram kernel will read from.

Purpose: Deliver QUANT-01's CPU half — `BinnedMatrix` + exact-quantile edges + searchsorted assignment — as a separately-testable component, NOT wired through `HistogramBackend` (Open-Question 2 resolved: standalone). The first task calibrates the A1 HIGH-risk f64/f32 cast point so edges are bit-equal to `numpy.quantile` before any golden vectors are frozen downstream.

Output: A new `crates/sylva-core/src/quantize/` module (mod/edges/binned_matrix/assign/report) re-exported from `lib.rs`, fully covered by Rust unit + proptest property tests.
</objective>
